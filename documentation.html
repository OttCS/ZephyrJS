ZephyrJS Documentation


Zephyr Application Structure

ZEPHYR
The full application
version
Number: The current version of Zephyr. the version is the date which it was released, in the format YYMMDD
mouse
Created when createMouseListener() is called, stores mouse data/functions
data
A map that stores data about the mouse (left/middle/right/x/y)
getX()
Returns the last known mouse x-coord. Called asynchronously outside the render loop.
RETURNS: Number, where 0.0-1.0 is inside the scene
getY()
Returns the last known mouse y-coord. Called asynchronously outside the render loop.
RETURNS: Number, where 0.0-1.0 is inside the scene
isDown(buttonString)
Returns true/false if the requested button (left, middle, right) is currently down
RETURNS: Boolean, True if the button is currently down, False if not
key
Created when createKeyListener() is called, stores key data/functions
data
A map that stores data about the keys (a, b, c, 1, 2, 3, shift, etc.)
isDown(keyString)
Returns true/false if the requested key ("a", "b", "1", "shift", etc.) is currently down
RETURNS: Boolean, True if the key is currently down, False if not
cacheMap
Map that stores images loaded with ZEPHYR.utils.cache(...), is used by Zephyr's render loop
layerMap
Map that stores the different "layers" (DOM canvases and other data) added with ZEPHYR.utils.addLayer(...) to use in the render loop
spriteMap
Map that stores sprites added with ZEPHYR.utils.setSprite()for access from the render loop
ticker
Set that stores the developer-added update functions to call every frame: Use the ZEPHYR.ticker.add((delta) => {}) function to add one (or more!)
utils
A set of commonly used functions for the user, which simplifies the developer's interactions with Zephyr
setTitle(documentTitle)
Sets the document title
setLayer(layerIdentifier, layerSettingsObject)
Adds a layer to the scene, or changes the settings of an existing one. All that is required is the name of the layer for reference by sprites and the desired settings (opacity, blend)
cache(imgURL)
Asynchronously creates an image and waits for it to successfully decode (while images can be "loaded" canvas blocks if they haven't "decoded" yet). Best called in a loading function, not in an update loop
setSprite(spriteIdentifier, spriteObject)
Creates a formatted sprite from the provided name and sprite-style object
NOTE: See the sprite structure below for what format to use for the sprite-style object
getSprite(spriteIdentifier)
Gets the stored sprite in memeory based on the sprite identifier
NOTE: If no sprite has been stored with the sprite identifier, null is returned
RETURNS: The stored sprite from ZEPHYR.spriteMap associated with the sprite identifier
createMouseListener()
Sets up and starts Zephyr's built-in asynchronous mouse listener
createKeyListener()
Sets up and starts Zephyr's built-in asynchronous keyboard listener
setViewCenter(coords)
Sets thev view center of the scene, like moving a camera
getPixelScale()
Gets an object with an x and y property, both having number values with the relative size of a pixel based on the height and width
RETURNS: An object with x and y properties
scene
Stores information about the scene
x
Number: The leftmost rendered pixel in the scene
y
Number: The topmost rendered pixel in the scene
width
Number: How wide the scene is, in rendered pixels
height
Number: How tall the scene is, in rendered pixels
smooth
Boolean: If pixel smoothing should occur
view
The DOM element (div) that holds the layered canvases
viewCenter
The relative center of the scene
x
The x coodinate of the center
DEFAULT: 0.5
y
The y coodinate of the center
DEFAULT: 0.5
system
Integral components to the Zephyr application
renderLoop()
The function that gets called to clear and redraw and the browser-allowed refresh rate.
delta
Stored millisecond time of the last render call
getSceneDOMBounds()
Gets and stores the document pixel offset of the scene
width
Stored DOM pixel width of the scene view
height
Stored DOM pixel height of the scene view
x
Stored left-edge DOM pixel of the scene view
y
Stored top-edge DOM pixel of the scene view
math
Functions focused around widely-used math stuff (collisions, etc.)
inScene(objectA)
RETURNS: True if the provided object is within the bounds of the current scene
collision(objectA, objectB)
Tests if the two objects are touching
RETURNS: True if the objects are touching, False if they are not touching
stat
Keeps track of statistics if the Zephyr application is started with the statistics flag
active
Boolean: True if stat monitoring is on, False if not
fps
Number: The current fps
ms
Number: How long it took to render the previous frame
element
The DOM output for the statistics
layerCulledSprites
The number of sprites that were on layers that were not updated
culledSprites
Number: The number of sprites outside the visible scene that were avoided
Application(settings)
The initial constructor for the Zephyr application, which handles almost all of the setup
appendScene(parent)
Appends Zephyr's scene to the specified parent element

Settings-Style Object Format

settings
Structure for the settings passed into the ZEPHYR.application(settings) constructor
NOTE: For the ZEPHYR.application(settings) constructor, all of these are optional. The default values are listed along with the property description.
width
Number: The pixel width of the scene
height
Number: The pixel height of the scene
smooth
Boolean: Whether the scene should use image smoothing
statistics
Boolean: Whether Zephyr should monitor and dsiplay statistics

Layer

layer
Stored layer structure inside ZEPHYR.layerMap. This is not used by developers, this is only documented for the sake of Zephyr development.
edited
Boolean: Stores if the layer has been edited (a sprite assigned to it has been changed)
element
The DOM canvas that represents the layer
ctx
The context for the canvas
minXDraw
Number: The lowest x value drawn to (for optimized clearing)
minYDraw
Number: The lowest y value drawn to (for optimized clearing
maxXDraw
Number: The highest x value drawn to (for optimized clearing
maxYDraw
Number: The highest y value drawn to (for optimized clearing

Sprite-Style Object Format

sprite
Structure for a sprite-style object. Zephyr's render loop can handle these.
NOTE: For ZEPHYR.utils.setSprite(spriteIdentifier, spriteObject), all of these are optional. The default values are listed along with the property description.
layer
String: The layer that the sprite will be drawn on
DEFAULT: ""
src
String: The image url/path to use.
DEFAULT: ""
NOTE: Make sure to cache this with ZEPHYR.utils.cache(...) before attempting to create the sprite, error will be thrown
x
Number: The x coordinate value (scene has a width of 1.0) of where the sprite should be drawn
DEFAULT: 0.0
y
Number: The y coordinate value (scene has a width of 1.0) of where the sprite should be drawn
DEFAULT: 0.0
anchor
The position that the sprite should be drawn relative to
x
Number: The x location of where on the sprite's image the position is relative to (0.0 is left edge, 1.0 is right edge)
DEFAULT: 0.0
y
Number: The y location of where on the sprite's image the position is relative to (0.0 is top edge, 1.0 is botton edge)
DEFAULT: 0.0
draw
Boolean: whether to draw the sprite or not
DEFAULT: false
cameraDependantPosition
Boolean: whether or not moving the scene view center should affect where this is drawn
DEFAULT: false
data
Stored data for Zephyr's render loop: x,y in real canvas pixels, width and height
NOTE: This is not an accepted parameter for ZEPHYR.utils.setSprite(...), data is auto-generated by Zephyr when the sprite is set or the scene view center is moved.
